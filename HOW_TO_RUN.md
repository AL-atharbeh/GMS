# دليل تشغيل مشروع نظام إدارة الكراجات (GMS) 🚀
# Garage Management System (GMS) - Run & Setup Guide

مرحباً! هذا الدليل يوضح خطوات إعداد وتشغيل المشروع وقاعدة البيانات الخاصة به بكل سهولة في كل مرة ترغب في تشغيله.

---

> [!IMPORTANT]
> **الروابط المباشرة للمشروع (المشروع يعمل حالياً على جهازك):**
> * 🌐 **رابط موقع المشروع (Frontend Web):** [http://localhost:3005](http://localhost:3005)
> * 🗄️ **رابط تصفح قاعدة البيانات (Prisma Studio):** [http://localhost:5555](http://localhost:5555)
> * 🔗 **رابط الواجهة الخلفية (Backend API):** [http://localhost:3001](http://localhost:3001)

---

## 🛠️ المتطلبات الأساسية (Prerequisites)
قبل البدء، تأكد من تثبيت البرامج التالية على جهازك:
1. **Node.js** (الإصدار 18 أو أحدث).
2. **Docker Desktop** (يجب أن يكون برنامج Docker شغّالاً في الخلفية).

---

## 🚀 الخطوة الأولى: إعداد قاعدة البيانات والبيانات الأساسية (للمرة الأولى فقط)
عند تشغيل المشروع لأول مرة، أو عند الرغبة في إعادة تهيئة قاعدة البيانات، قم باتباع الخطوات التالية:

1. تأكد من أن تطبيق **Docker Desktop** مفتوح وشغّال.
2. افتح مبنى الأوامر (Terminal) في المجلد الرئيسي للمشروع، وقم بتشغيل ملف الإعداد التلقائي:
   ```bash
   chmod +x setup.sh
   ./setup.sh
   ```
   *يقوم هذا الملف بـ:*
   - تشغيل قواعد البيانات (PostgreSQL و Redis) داخل حاويات Docker.
   - تطبيق الهجرات (Migrations) لإنشاء الجداول في قاعدة البيانات.
   - توليد عميل Prisma (Prisma Client).
   - إدخال البيانات التجريبية الأساسية (Seeding).

> [!NOTE]
> إذا واجهت مشكلة تفاعلية مع الـ Migrations أثناء التشغيل التلقائي، يمكنك تشغيل هذه الأوامر يدوياً لتهيئة قاعدة البيانات:
> ```bash
> cd apps/api
> npx prisma migrate deploy
> npx prisma generate
> npx tsx src/prisma/seed.ts
> cd ../..
> ```

---

## 💻 الخطوة الثانية: تشغيل المشروع (Frontend & Backend)
لتشغيل الواجهة الأمامية والخلفية معاً بطلب أمر واحد من المجلد الرئيسي للمشروع:

```bash
npm run dev
```

عند تشغيل هذا الأمر، سيعمل المشروع كالتالي:
* **الواجهة الخلفية (Backend API):** ستكون متاحة على الرابط `http://localhost:3001`
* **رابط فحص الحالة (Health Check):** `http://localhost:3001/health`
* **الواجهة الأمامية (Frontend Web):** ستكون متاحة على الرابط `http://localhost:3005`

## 🗄️ تصفح قاعدة البيانات بصرياً (Prisma Studio)
إذا كنت تريد تصفح جداول قاعدة البيانات وتعديل البيانات مباشرة من المتصفح، يمكنك تشغيل الأداة التالية من المجلد الرئيسي للمشروع:

```bash
npm run db:studio
```
ثم افتح الرابط الذي سيظهر لك في المتصفح (غالباً `http://localhost:5555`).

---

## 🛑 إيقاف قواعد البيانات
عند الانتهاء من العمل وتريد إيقاف خدمات قاعدة البيانات والـ Redis لتوفير موارد الجهاز، قم بتشغيل الأمر التالي:

```bash
docker compose down
```
